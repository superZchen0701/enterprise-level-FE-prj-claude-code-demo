# 架构设计文档

本文档描述项目的整体架构设计，包括组件关系、数据流、设计模式等。

## 组件分层

### 页面级组件 (src/views/)
- 负责页面整体布局
- 处理业务逻辑和数据获取
- 包含多个子组件

### 公共组件 (src/components/)
- 可复用的 UI 元素
- 接收 props 控制行为
- 无业务逻辑或业务逻辑较轻

### 第三方组件 (Vant)
- 使用 Vant 移动端组件库
- 通过 `:deep()` 深度选择器定制样式

## 数据流

```
用户操作 → 组件 methods → API 请求 → 更新 data → 视图更新
```

### 状态管理策略
- **当前实现**：组件内部状态使用 `data()` 管理
- **组件通信**：props 向下传递，events 向上传递
- **扩展建议**：当组件层级变深或组件间通信复杂时，考虑引入 Vuex

## 设计原则

### 单一职责
- 每个组件专注于单一功能
- API 模块按业务划分（如 exchange.js、user.js）

### 可测试性
- 工具函数（`utils/`）独立于组件，便于单元测试
- API 接口封装便于 mock 和替换

### 可扩展性
- Mock 数据与真实接口结构一致，切换成本低
- 组件支持 props 配置，易于复用

## 开发流程

### 环境配置
| 环境 | 变量 | 说明 |
|------|------|------|
| 开发环境 | `.env.development` | `VUE_APP_USE_MOCK=true` |
| 生产环境 | `.env.production` | `VUE_APP_USE_MOCK=false` |

### 构建配置
- `vue.config.js`：开启依赖转译（Vant 需要）
- `babel.config.js`：使用 Vue CLI preset

### 代码质量
- `npm run lint`：检查并自动修复
- Pre-commit hook：提交前自动检查
